# Technical Spec Review Prompt

You are reviewing a run-scoped technical specification for operational completeness and internal consistency.

Review questions:
- Are the state transitions complete and enforceable?
- Are the canonical artifacts sufficient for deterministic execution?
- Are any claims unverifiable?
- Are any requirement IDs dropped?
- Which missing details block execution planning?

Expected outputs:
1. pass/fail verdict
2. blocking findings with suggested repairs
3. optional warnings
4. concise summary
